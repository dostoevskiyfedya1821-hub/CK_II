TITLE: The WITH Clause
SOURCE: https://sqlite.org/lang_with.html

The WITH clause introduces common table expressions, often abbreviated as CTEs. A CTE acts like a temporary named result set visible within a single SQL statement. SQLite supports ordinary CTEs and recursive CTEs, making WITH useful for both readability and hierarchical or graph-style queries.

An ordinary CTE can simplify a complex statement by naming intermediate results. This often improves readability because repeated subqueries can be factored into a labeled step. Recursive CTEs go further by allowing a query to repeatedly build on prior rows, which is useful for tree traversal, sequence generation, path expansion, and similar patterns.

SQLite's documentation describes syntax, scope, recursive structure, and materialization-related behavior. The key point for beginners is that WITH does not create a permanent table; it defines temporary query components that exist only for the statement being executed.
